# Implementation Plan: csv-persistent-storage

## Overview

Replace all `localStorage.getItem` / `localStorage.setItem` calls in the MMS with a thin
`CsvStore` abstraction that persists data as RFC 4180 CSV files in monthly sub-folders.
The implementation is purely additive: new files (`config-loader.js`, `csv-store.js`,
`io-adapter.js`) are created, and the minimal two-line change per page script is wired in at
the end. All existing HTML, CSS, and event-handler code is left untouched.

Implementation language: **JavaScript** (ES modules, browser).
Test framework: **Vitest** with **fast-check** for property-based tests.

## Notes

- Tasks marked with `*` are optional and can be skipped for a faster MVP.
- Each task references specific requirements for traceability.
- Checkpoints ensure incremental validation at each major layer boundary.
- Property tests (Properties 1–15) validate universal correctness guarantees; unit tests validate
  specific examples and edge cases.
- All new UI elements (identity modal, migration dialog, toast, banners) use `position: fixed` or
  `position: absolute` so no existing page element is shifted or reflowed (Requirement 10.4).
- `io-adapter.js` contains exactly two adapters: `FsapiAdapter` and `LocalStorageAdapter`.
  There is no companion server, no `CompanionAdapter`, and no server-side runtime of any kind.
- `config.json` has exactly four keys: `dataPath`, `appTitle`, `defaultOperatingHours`,
  `lockTimeoutSeconds`. Property 7 validates all four keys.
